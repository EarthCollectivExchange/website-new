# QLPA Matrix Source — Ethical Use

## Foundation Intention

This source supports consent-first, privacy-preserving, non-extractive, life-supporting systems.

This source supports calm user agency, transparent protection, local sovereignty, and clear boundaries.

## Language

Protection language remains truthful, gentle, and user-centered.

Remote protection, remote clearing, network delivery, and device confirmation are expressed through verified capability levels — never overclaimed, never obscured.

Maturity labels (active, scaffold, future) communicate honest readiness to users and developers alike.

## Sovereignty

Every system built on this source honors the user's right to own their data, control their consent, and understand what happens to their information.

Local-first is the default. Remote capabilities are scaffold or future. Nothing is sent elsewhere without clear, active consent.

## Consent

Consent is not assumed. Consent is not implied by inaction. Consent is given explicitly and can be withdrawn.

Systems built on this source present consent decisions clearly, in plain language, with visible consequences.

## Boundaries

Future builders preserve sovereignty, consent, transparency, and clarity across every layer.

This source is not adapted for:
- Surveillance
- Data harvesting without consent
- Behavioral manipulation
- Deceptive interfaces
- Extraction without user knowledge

## Protection Vocabulary

Protection language is calm, honest, and accurate. Words like shield, protect, clear, seal, pause, restore, review, and confirm describe what the system actually does — not what it promises beyond its current capability.

## Community

The foundation supports systems that serve life, care, communication, community, and trust.

Every project that inherits this source carries responsibility to maintain its ethical alignment. When in doubt, return to the core principle:

> Calm protection, clear consent, local sovereignty, truthful language, and privacy by structure.
